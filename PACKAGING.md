# Packaging a release archive

This document explains how to create the ZIP file that ordinary users should download.

A user release archive should contain the compiled plugin DLL and texture override files only. It should not contain source code or build scripts.

## Release archive structure

The release ZIP should extract into the game folder and produce this structure:

```text
BepInEx/
  plugins/
    ITRTGTextureOverrideMod.dll
    ITRTGTextureOverrides/
      README.txt
      texture_overrides.tsv
      Textures/
        food_puny.png
        food_strong.png
        food_mighty.png
        rebirthbacon.png
        fsm.png
        fsm_evo.png
        stick_fsm.png
```

## Files to include

Include:

```text
ModPackage/BepInEx/plugins/ITRTGTextureOverrideMod.dll
ModPackage/BepInEx/plugins/ITRTGTextureOverrides/
```

Optionally include a top-level `README.md` or `README.txt` in the ZIP, but keep the `BepInEx/plugins/...` structure intact.

## Files not to include

Do not include:

```text
src/
build.sh
install_to_game.sh
*.csproj generated by your IDE
Assembly-CSharp.dll
Assembly-CSharp.zip
UnityEngine*.dll
BepInEx.dll
0Harmony.dll
resources.assets
sharedassets*.assets
```

The user must install BepInEx separately. Do not redistribute BepInEx binaries inside this mod archive unless you have intentionally prepared a combined installer and checked the relevant license terms.

Do not redistribute game assemblies or game assets.

## Linux packaging command

From the repository root, after building the DLL:

```bash
cd ModPackage
zip -r ../ITRTG_Vegan_Texture_Mod.zip BepInEx
```

The resulting file is:

```text
ITRTG_Vegan_Texture_Mod.zip
```

Users should extract it into the game folder.

## Packaging directly from an installed game folder

If the compiled DLL exists only in your local game folder, you can package from there:

```bash
GAME_DIR="$HOME/.steam/debian-installation/steamapps/common/Idling to Rule the Gods"
OUT_DIR="$HOME/Downloads/ITRTG_Vegan_Texture_Mod"

rm -rf "$OUT_DIR"
mkdir -p "$OUT_DIR/BepInEx/plugins"

cp "$GAME_DIR/BepInEx/plugins/ITRTGTextureOverrideMod.dll" \
   "$OUT_DIR/BepInEx/plugins/"

cp -r "$GAME_DIR/BepInEx/plugins/ITRTGTextureOverrides" \
   "$OUT_DIR/BepInEx/plugins/"

cd "$HOME/Downloads"
zip -r ITRTG_Vegan_Texture_Mod.zip ITRTG_Vegan_Texture_Mod
```

## Recommended GitHub release assets

Attach at least two ZIP files to a GitHub release:

1. `ITRTG_Vegan_Texture_Mod.zip` — ready-made user package.
2. `Source code` — GitHub can generate this automatically from the tag.

Optionally attach:

```text
ITRTG_Vegan_Texture_Mod_TexturesOnly.zip
```

for users who already have the plugin DLL and only want to update the art.

## Versioning suggestion

Use simple semantic-like versions:

```text
v1.0.0 - initial public release
v1.1.0 - new texture replacements
v1.1.1 - bugfix / path fix
```

Update `CHANGELOG.md` when publishing a new release.
